A. Why Git/GitHub were needed?
Imagine a team building a large software system. Multiple developers edit the same files, often simultaneously.
 Before modern version control, teams used manual file copies like final_v2_latest_revised_REAL.py. 
 This caused conflicts, lost work, no history tracking, and poor collaboration. 
 If a bug appeared, tracing who changed what—and reverting safely—was extremely difficult. 
 Teams needed a systematic way to track changes, collaborate concurrently, and maintain a reliable history.

B. What is Git?

Git is a distributed version control system (DVCS). It tracks changes in code over time, allowing developers to:
1. Record snapshots (commits)
2. Work independently via branches
3. Merge changes efficiently
4. Revert to previous states when needed

Because it’s distributed, every developer has a full copy of the project history, improving resilience and speed.

C. What is GitHub?

GitHub is a cloud-based platform built around Git. It adds:
1. Centralized repositories for collaboration
2. Access control and team management
3. Pull requests for code review
4. Issue tracking and CI/CD integrations
In short: Git = tool for version control, GitHub = platform for collaboration using Git.

Commands for GitHub:

Core Git Commands (with practical usage)

Note: These commands are part of Git, and commonly used with platforms like GitHub.

1. Initialize a Repository
git init
Use case: Start version control in a project folder
Example:
cd my-project
git init

2. Check Status
git status
Use case: See modified, staged, or untracked files

3. Add Files to Staging Area
A. git add .
Use case: Add all changes

B. git add file_name
Use case: Add a specific file

4. Commit Changes
git commit -m "message"
Use case: Save a snapshot of staged changes

5. Connect to GitHub Repository
git remote add origin <repo_url>
Example:
git remote add origin https://github.com/user/repo.git

6. Push Code to GitHub
git push origin main
Use case: Upload local commits to GitHub

7. Pull Latest Changes
git pull origin main
Use case: Fetch and merge updates from GitHub

8. Clone a Repository
git clone <repo_url>
Example:
git clone https://github.com/user/repo.git

9. Create and Switch Branch
git checkout -b feature-branch
Use case: Work on a new feature without affecting main code

10. Merge Branch
git merge feature-branch
Use case: Combine changes into current branch

11. View Commit History
git log
